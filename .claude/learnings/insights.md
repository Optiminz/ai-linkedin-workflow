# Project Insights

## 2026-03-11: Guide Tone — Explain, Don't Instruct

When writing guides for AI-assisted workflows, frame documentation as "what happens and why" rather than "how to do it manually." Users reading instructional language ("open this file," "edit this section," "create files in this directory") overestimate the effort required when the AI handles the mechanics. The guides should be educational context that helps users make better decisions, not step-by-step manuals.

**Before:** "Open `EDITORIAL-CONSTITUTION.md` and find the Voice Dimensions section. Set these three scales:"
**After:** "During setup, Claude Code analyzes your writing samples and sets three voice dimensions automatically."

Also: non-technical users seeing "clone a repo" already feel intimidated. An early "there's no code in here" reassurance + explanation of why it's a repo (visible files you control vs hidden settings) significantly lowers the barrier.
